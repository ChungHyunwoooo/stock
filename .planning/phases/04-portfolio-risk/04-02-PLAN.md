1. engine/strategy/portfolio_risk.py 신규 생성:

```python
@dataclass
class PortfolioRiskConfig:
    enabled: bool = True
    correlation_threshold: float = 0.7      # 기본 상관관계 차단 임계치
    correlation_window: int = 100           # 상관관계 계산 윈도우 (봉 수)
    strategy_overrides: dict[str, float] = field(default_factory=dict)  # {strategy_id: threshold}
    risk_parity_config: RiskParityConfig = field(default_factory=RiskParityConfig)
```

2. PortfolioRiskManager class:
   - __init__(config: PortfolioRiskConfig | None = None, notifier: NotificationPort | None = None)
   - _active_signals: dict[str, pd.Series] = {} — {strategy_id: recent_returns_series}
   - _allocation_weights: dict[str, float] = {}

   - register_strategy(strategy_id: str, returns: pd.Series) -> None
     활성 전략 등록 (수익률 시계열 저장)

   - unregister_strategy(strategy_id: str) -> None
     전략 제거 + 배분 재계산

   - check_correlation_gate(strategy_id: str, signal_returns: pd.Series) -> tuple[bool, str]
     - enabled=False이면 (True, "gate_disabled") 반환
     - 등록된 활성 전략 없으면 (True, "no_active_strategies") 반환
     - signal_returns와 각 활성 전략 returns의 pearson 상관계수 계산
     - 전략별 오버라이드 임계치 또는 기본 0.7 적용
     - 0.7 초과 발견 시 (False, f"blocked: corr={corr:.3f} with {other_id}, threshold={threshold}")
     - 차단 시 logger.info 기록 + notifier.send_text() (notifier 있으면)
     - 통과 시 (True, "passed")

   - get_allocation_weights() -> dict[str, float]
     캐시된 _allocation_weights 반환 (없으면 균등 배분)

   - refresh_weights() -> dict[str, float]
     _active_signals의 수익률로 calculate_risk_parity_weights() 호출 → _allocation_weights 갱신

3. 상관계수 계산: pd.Series.corr() 사용 (multi_symbol.py 패턴 참고). 공통 인덱스가 correlation_window 미만이면 가용 데이터 전체 사용. 데이터 부족(< 10봉) 시 상관관계 0으로 간주 (허용).
