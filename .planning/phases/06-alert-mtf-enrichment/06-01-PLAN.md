## Objective

매매 체결, 전략 상태 변화, 시스템 이상, 백테스트 완료 4가지 이벤트를 각각 구분된 Discord embed 메시지로 실시간 발송한다.

Purpose: MON-01 — 실매매 운영 중 중요 이벤트를 놓치지 않고 Discord로 즉시 확인
Output: EventNotifier 모듈 + LifecycleManager 콜백 연동 + 테스트

## Tasks

<task id="1">
<title>EventNotifier 모듈 생성 + 4가지 이벤트 embed 포매터</title>
<action>
engine/notifications/event_notifier.py 생성:

1. `EventNotifier` 클래스 — NotificationPort를 감싸는 래퍼:
   - `__init__(self, notifier: NotificationPort)` — 기존 NotificationPort 주입
   - `notify_execution(execution: ExecutionRecord)` — 체결 embed (심볼, 방향, 수량, 가격, 브로커, 시간)
   - `notify_lifecycle_transition(strategy_id: str, from_status: str, to_status: str, reason: str = "")` — 전이 embed (전략ID, 이전→이후 상태, 사유)
   - `notify_system_error(component: str, error: str, severity: str = "WARNING")` — 시스템 이상 embed (컴포넌트명, 에러 메시지, WARNING/CRITICAL 색상 구분)
   - `notify_backtest_complete(strategy_id: str, symbol: str, sharpe: float | None, total_return: float, max_dd: float | None)` — 백테스트 결과 embed

2. 각 메서드는 내부적으로 `self.notifier.send_text(formatted_message)`를 호출하여 기존 NotificationPort 변경 없이 동작.

3. 메시지 포맷은 Discord embed markdown 활용:
   - 체결: "[EXECUTION] BTC/USDT LONG 0.5 @ 65,000 (paper)"
   - 전이: "[LIFECYCLE] rsi_divergence: testing -> paper"
   - 시스템: "[WARNING] ScalpingRunner: WebSocket timeout after 30s"
   - 백테스트: "[BACKTEST] rsi_divergence BTC/USDT — Sharpe 1.23, Return +15.2%, MaxDD -8.1%"

4. tests/test_event_notifier.py:
   - MemoryNotifier를 주입하여 각 이벤트 메서드 호출 후 messages 리스트에 올바른 포맷 문자열이 추가되는지 검증
   - 4개 이벤트 x 정상 케이스 = 최소 4개 테스트
</action>

<done_when>pytest tests/test_event_notifier.py 통과, 4개 이벤트 타입 모두 MemoryNotifier에 올바른 포맷으로 기록됨</done_when>
</task>

<task id="2">
<title>LifecycleManager 전이 콜백 + Orchestrator 체결 알림 연동</title>
<action>
1. engine/strategy/lifecycle_manager.py 수정:
   - `on_transition` 콜백 리스트 속성 추가: `self._on_transition_callbacks: list[Callable[[str, str, str], None]] = []`
   - `add_transition_listener(callback: Callable[[str, str, str], None])` 메서드 추가
   - `transition()` 메서드 내 성공적 전이 후 `for cb in self._on_transition_callbacks: cb(strategy_id, from_status.value, to_status.value)` 호출
   - 기존 동작 변경 없음 — 콜백 미등록 시 아무 일 없음

2. engine/application/trading/orchestrator.py 수정:
   - `__init__`에 `event_notifier: EventNotifier | None = None` 파라미터 추가 (기본 None, 하위 호환)
   - `process_signal()` 내 체결 성공 후 `if self.event_notifier: self.event_notifier.notify_execution(execution)` 추가
   - 기존 `self.notifier.send_execution(execution)` 유지 — event_notifier는 추가 알림 채널

3. engine/interfaces/bootstrap.py에서 EventNotifier 생성 + LifecycleManager 콜백 등록은 Phase 6 범위 밖 (개별 실행 환경에서 조립). 대신 tests/test_event_notifier.py에 통합 시나리오 테스트 추가:
