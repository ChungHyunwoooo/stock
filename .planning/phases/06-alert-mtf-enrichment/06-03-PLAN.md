## Objective

상위 타임프레임(예: 4h, 1d) 방향을 확인하여 단기 타임프레임 진입 신호를 필터링하는 MTF confirmation gate를 구현한다. 설정으로 활성화/비활성화 가능.

Purpose: MON-04 — 단기 노이즈 신호를 상위 TF 추세 방향으로 필터링하여 진입 품질 향상
Output: MTFConfirmationGate 모듈 + Orchestrator 연동 + 테스트

## Tasks

<task id="1">
<title>MTFConfirmationGate 모듈 구현</title>
<action>
1. engine/strategy/mtf_filter.py 생성:

   ```python
   @dataclass
   class MTFConfig:
       enabled: bool = False  # 기본 비활성화
       higher_timeframe: str = "4h"  # 상위 TF (4h, 1d 등)
       ema_period: int = 20  # 방향 판단용 EMA 기간
       lookback_bars: int = 50  # 상위 TF 봉 조회 수
   ```

2. `MTFConfirmationGate` 클래스:
   - `__init__(self, config: MTFConfig, data_provider: DataProvider | None = None)`
   - config.enabled가 False이면 모든 check_alignment()이 (True, "MTF filter disabled") 반환
   - data_provider가 None이면 (True, "No data provider") 반환

   - `check_alignment(self, symbol: str, side: TradeSide, signal_timeframe: str) -> tuple[bool, str]`:
     a. signal_timeframe >= higher_timeframe이면 MTF 불필요 → (True, "Signal TF >= Higher TF")
     b. data_provider.fetch_ohlcv(symbol, start, end, higher_timeframe)로 상위 TF 봉 조회
        - start = 현재 시점에서 lookback_bars * higher_tf_duration만큼 이전
        - 조회 실패 시 (True, "Data fetch failed, allowing signal") — 데이터 실패가 매매를 차단하면 안 됨
     c. EMA(ema_period) 계산 (engine/indicators/price/ 기존 EMA 함수 활용 또는 pandas ewm 직접 사용)
     d. 방향 판단:
        - 현재가 > EMA → higher_tf_direction = "LONG"
        - 현재가 < EMA → higher_tf_direction = "SHORT"
     e. 정렬 확인:
        - side == higher_tf_direction → (True, f"Aligned with {higher_timeframe} trend")
        - side != higher_tf_direction → (False, f"Against {higher_timeframe} trend (EMA direction: {higher_tf_direction})")

   - `_timeframe_to_minutes(tf: str) -> int` 헬퍼: "1m"->1, "5m"->5, "15m"->15, "1h"->60, "4h"->240, "1d"->1440

3. MTFConfig 로딩: `config/trading.json`의 `mtf_filter` 키에서 로드. 파일 없으면 기본값(disabled).
   ```python
   @classmethod
   def from_config(cls) -> MTFConfig:
       path = config_file("trading.json")
       if not path.exists():
           return cls()
       data = json.loads(path.read_text())
       mtf = data.get("mtf_filter", {})
       return cls(**{k: v for k, v in mtf.items() if k in cls.__dataclass_fields__})
   ```

4. tests/test_mtf_filter.py:
   - MTFConfig(enabled=False) → check_alignment() 항상 True
   - MTFConfig(enabled=True) + mock data_provider:
     - 현재가 > EMA + side=LONG → True (정렬)
     - 현재가 > EMA + side=SHORT → False (반대)
     - 현재가 < EMA + side=SHORT → True (정렬)
     - 현재가 < EMA + side=LONG → False (반대)
   - data_provider=None → True (graceful)
   - fetch_ohlcv 예외 → True (데이터 실패 시 허용)
   - signal_timeframe >= higher_timeframe → True (MTF 불필요)
