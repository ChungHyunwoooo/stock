## Objective

Discord /status 커맨드를 확장하여 현재 포지션, 일일 PnL, 전략별 상태를 한 번에 조회할 수 있게 한다. 응답은 5초 이내.

Purpose: MON-02 — 모바일 Discord에서 시스템 상태를 즉시 파악
Output: StatusCommandPlugin + 포매팅 함수 + 테스트

## Tasks

<task id="1">
<title>StatusCommandPlugin + 포매팅 함수 구현</title>
<action>
1. engine/interfaces/discord/formatting.py 확장 — 새 함수 추가:
   - `format_status_embed(control: TradingControlService, lifecycle: LifecycleManager) -> str`:
     a. 런타임 섹션: mode, paused, automation_enabled
     b. 포지션 섹션: state.positions에서 status=open인 것들 → 심볼, 방향, 진입가, 현재 PnL% 리스트
     c. 일일 PnL 섹션: state.executions에서 오늘(UTC) 체결 건의 실현 PnL 합산. 체결 건이 없으면 "No trades today"
     d. 전략 상태 섹션: lifecycle.list_strategies()로 전략 목록 조회 → status별 카운트 (active: N, paper: N, testing: N, draft: N)
     e. 마지막 업데이트 시각
   - 포맷은 Discord markdown 코드블록 활용하여 정렬된 테이블 형태

2. engine/interfaces/discord/commands/status.py 생성:
   - `StatusCommandPlugin` 클래스 (DiscordCommandPlugin 프로토콜 준수)
   - `name = "status_v2"`
   - `register(tree, context)`:
     - 기존 RuntimeCommandPlugin의 /status를 대체하지 않고 /상태 (한글 커맨드) 또는 /dashboard 이름으로 등록
     - 실제로는 기존 /status 커맨드를 이 플러그인으로 교체: RuntimeCommandPlugin에서 /status 제거하고 StatusCommandPlugin이 /status 등록
   - `interaction.response.defer()` 호출 후 데이터 조회 → `interaction.followup.send()` 패턴으로 5초 제한 회피

3. engine/interfaces/discord/commands/__init__.py 수정:
   - `StatusCommandPlugin` import 추가
   - `DEFAULT_COMMAND_PLUGINS`에 추가
   - RuntimeCommandPlugin에서 /status 제거 (RuntimeCommandPlugin은 /pause, /resume, /mode만 유지)

4. engine/interfaces/discord/commands/runtime.py 수정:
   - /status 커맨드 등록 제거 (StatusCommandPlugin으로 이관)

5. tests/test_status_command.py:
   - format_status_embed() 단위 테스트:
     - 포지션 있는 경우 심볼/방향/PnL 표시 확인
     - 포지션 없는 경우 "No open positions" 표시 확인

   - Mock TradingControlService + LifecycleManager 사용
