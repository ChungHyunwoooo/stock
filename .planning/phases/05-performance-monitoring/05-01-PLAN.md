Output: `engine/strategy/performance_monitor.py` + per-strategy pause 지원 + 테스트

- TradingRuntimeState.paused_strategies: set[str] 필드 추가, 기본값 빈 set
- JsonRuntimeStore가 paused_strategies를 직렬화/역직렬화 (set -> list -> set)
- TradingOrchestrator.process_signal()에서 signal.strategy_id가 paused_strategies에 있으면 신호 무시 + 알림
- PerformanceConfig: warning_threshold=0.15, critical_sharpe=-0.5, rolling_window=20, rolling_window_extended=30, check_interval_seconds=900
- PerformanceSnapshot: strategy_id, rolling_sharpe, rolling_win_rate, baseline_sharpe, baseline_win_rate, degradation_pct_sharpe, degradation_pct_win_rate, alert_level (none/warning/critical)
- StrategyPerformanceMonitor.__init__(trade_repo, backtest_repo, lifecycle, runtime_store, notifier, config)
- _compute_rolling_metrics(trades: list[TradeRecord], window: int) -> (sharpe, win_rate) -- profit_pct 기반 raw Sharpe (mean/std), 최소 window건 미만이면 None 반환
- _get_baseline(session, strategy_id) -> (sharpe, win_rate) -- BacktestRecord.get_history limit=1의 sharpe_ratio + result_json에서 win_rate 추출, 없으면 None
- _evaluate_strategy(session, strategy_id) -> PerformanceSnapshot -- 20거래 윈도우 Sharpe/승률 계산 + baseline 비교 + alert_level 판정
- check_all(session) -> list[PerformanceSnapshot] -- active 전략 순회, _evaluate_strategy 호출
- WARNING: baseline 대비 Sharpe 또는 승률이 15% 이상 하락
- CRITICAL: 30거래 롤링 Sharpe < -0.5
- run_daemon() -- 데몬 스레드로 check_interval_seconds 주기 실행, try/except로 개별 에러 격리
</behavior>
<action>
1. `engine/core/models.py`: TradingRuntimeState에 `paused_strategies: set[str] = field(default_factory=set)` 추가

2. `engine/core/json_store.py`: JsonRuntimeStore 수정
   - `_state_to_dict`: paused_strategies를 `list(state.paused_strategies)`로 직렬화
   - `_state_from_dict`: `set(data.get("paused_strategies", []))`로 역직렬화

3. `engine/application/trading/orchestrator.py`: process_signal() 상단 paused 체크 블록 바로 아래에 per-strategy 체크 추가
   ```python
   if signal.strategy_id in state.paused_strategies:
       self.notifier.send_text(
           f"Signal {signal.signal_id} for {signal.symbol} skipped: "
           f"strategy {signal.strategy_id} is paused (performance degradation)."
       )
       self.runtime_store.save(state)
       return state
   ```

4. `engine/strategy/performance_monitor.py`: 새 파일 생성
   - PerformanceConfig dataclass (warning_threshold, critical_sharpe, rolling_window, rolling_window_extended, check_interval_seconds)
   - PerformanceSnapshot dataclass (strategy_id, rolling_sharpe, rolling_win_rate, baseline_sharpe, baseline_win_rate, degradation_pct_sharpe, degradation_pct_win_rate, alert_level)
   - StrategyPerformanceMonitor 클래스:
     - 생성자: trade_repo, backtest_repo, lifecycle, runtime_store, notifier, config 주입
     - _compute_rolling_metrics: numpy 없이 순수 파이썬으로 mean/std 계산, window 미만이면 (None, None) 반환
     - _get_baseline: BacktestRepository에서 최신 백테스트 sharpe_ratio + result_json 파싱하여 win_rate 추출
     - _evaluate_strategy: 20거래 윈도우 Sharpe/승률 vs baseline, 30거래 Sharpe < -0.5 체크
     - check_all: LifecycleManager.list_by_status("active")로 대상 전략 목록 조회, 순회하며 _evaluate_strategy
     - run_daemon: threading.Thread(daemon=True) 시작, while True 루프에서 check_interval_seconds sleep 후 check_all 실행, 개별 전략 에러는 logger.exception으로 격리

5. `tests/test_performance_monitor.py`: TDD 테스트
