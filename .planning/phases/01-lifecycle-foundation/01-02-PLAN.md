  <name>Task 2: API 라우터에 LifecycleManager 연동</name>
  <files>api/routers/strategies.py</files>
  <action>
api/routers/strategies.py의 update_status 엔드포인트를 수정하여 LifecycleManager를 통한 전이 검증을 추가한다.

주의: 기존 API는 DB(StrategyRecord)를 사용한다. CONTEXT.md 결정에 따라 "API 전략 조회는 registry.json을 직접 읽어서 응답"이지만, 이 API 라우터는 DB 기반이므로 Phase 1에서는 최소 변경만 적용한다:

1. PATCH /{strategy_id}/status 핸들러에 LifecycleManager 검증 추가:
   - LifecycleManager를 import하고 인스턴스 생성 (모듈 레벨 또는 함수 내)
   - body.status로 전이를 시도하기 전에 registry.json의 해당 전략 status를 확인
   - registry.json에 해당 strategy가 없으면 기존 DB 로직 유지 (기존 전략은 상태머신 대상 아님)
   - registry.json에 있는 전략이면 LifecycleManager.transition() 호출하여 전이 검증
   - InvalidTransitionError 발생 시 HTTPException(400, detail=str(e)) 반환
   - 성공 시 기존 DB update도 수행 (양쪽 동기화)

2. try/except로 감싸서 LifecycleManager 관련 에러가 API 응답으로 변환되게 한다.

이 변경은 기존 API 동작을 깨뜨리지 않는다: registry.json에 없는 전략은 기존 로직 그대로 동작한다.
