  <name>Task 1: CPCVValidator 구현 + 테스트</name>
  <files>engine/backtest/cpcv.py, tests/test_cpcv.py</files>
  <behavior>
    - validate(equity_curve) returns ValidationResult with mode="cpcv"
    - Default: n_folds=6, n_test_folds=2, purged_size=10, embargo_size=5, gap_threshold=0.5
    - C(6,2) = 15 paths generated
    - Each path: train on (n_folds - n_test_folds) folds, test on n_test_folds
    - Purging: purged_size bars removed between train/test boundary
    - Embargo: embargo_size bars skipped after test fold
    - overall_passed = pass_rate >= 0.5 (50% 이상 경로 통과)
    - summary contains n_paths and pass_rate
    - Monotonically increasing curve => high pass_rate
    - Edge case: too short curve => ValueError
  </behavior>
  <action>
    1. `engine/backtest/cpcv.py`:
       - CPCVValidator 클래스: __init__(n_folds=6, n_test_folds=2, purged_size=10, embargo_size=5, gap_threshold=0.5)
       - validate(equity_curve: pd.Series) -> ValidationResult
       - skfolio 설치 여부 확인 — 있으면 skfolio.model_selection.CombinatorialPurgedCV 사용, 없으면 fallback 구현
       - skfolio 사용 시: CombinatorialPurgedCV(n_folds, n_test_folds, purged_size, embargo_size).split(X) 호출
       - 각 split에서 train_idx/test_idx 추출, IS/OOS returns 계산, Sharpe 비교
       - gap_ratio = oos_sharpe / is_sharpe, passed = gap_ratio >= gap_threshold
       - overall_passed = pass_rate >= 0.5
       - _calc_sharpe 내부 메서드 (WalkForwardValidator와 동일 로직)
       - too short면 ValueError
       - RESEARCH.md CPCV 코드 예시 참조 (단, test_indices 구조는 skfolio 실제 API에 맞게 조정)

    2. `tests/test_cpcv.py`:
       - 단조 증가 equity curve -> high pass_rate 테스트
       - mode == "cpcv" 확인
       - summary에 n_paths, pass_rate 포함 확인
       - paths 개수 > 0 확인
       - too short curve -> ValueError 테스트
       - WalkForwardValidator와 동일 equity curve에 대해 둘 다 ValidationResult 타입 반환 확인

    NOTE: skfolio가 pip install 필요 — 테스트 실행 전 `.venv/bin/pip install skfolio` 실행. RESEARCH.md에 명시된 유일한 신규 의존성.
